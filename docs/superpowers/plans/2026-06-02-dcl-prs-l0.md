# DCL-PRS L0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the first executable evidence chain for DCL-PRS: shared gate skeleton, cross-domain public dataset ingestion audit, mechanism program artifact, failure attribution artifact, and a constrained cohort simulation trace.

**Architecture:** Follow the existing `experiments/*.py` artifact-first pattern. Each DCL-PRS component emits a strict JSON artifact with schema version, claim boundary, risk flags, and CLI writer; the integrated gate consumes those artifacts and keeps CCF-A/Product claims open until evidence is sufficient.

**Tech Stack:** Python standard library, `pytest`, local JSON fixtures/artifacts, existing `experiments/results/` convention.

---

## File Structure

- Create `experiments/dcl_prs_public_dataset_ingestion.py`
  - Owns GSS / Eurobarometer / WVS source audit metadata, task mapping smoke, and CLI artifact writing.
- Create `experiments/dcl_prs_mechanism_program.py`
  - Owns structured mechanism program schema and first deterministic mechanism generation for current policy tasks.
- Create `experiments/dcl_prs_failure_attribution.py`
  - Owns error taxonomy, deterministic attribution from failed segment diagnostics, and repair candidate skeletons.
- Create `experiments/dcl_prs_dynamic_simulation.py`
  - Owns constrained cohort-level simulation trace from mechanism programs.
- Create `experiments/dcl_prs_gate_index.py`
  - Owns integrated gate over Task 1-5 artifacts, route coverage ledger, claim boundary, paper gate status, and product readiness status.
- Create tests:
  - `tests/test_dcl_prs_public_dataset_ingestion.py`

  - `tests/test_dcl_prs_failure_attribution.py`
